Introduction

The InTTO Automated Certificate System is a web-based platform designed to streamline the generation, management, and distribution of event certificates. This documentation details the system architecture, installation procedures, usage workflows, and API endpoints. It is intended for administrators, developers, and end-users seeking to understand or integrate with the certificate automation process for InTTO events.

Usage

Admin Panel

  • Log in to the admin panel.
  • Create a new event.
  • Add questions for the participants.
  • Upload a certificate template.
  • Deploy the feedback form with a unique QR code.
  • View feedback ratings, comments and suggestions of current or previous events.

User Feedback & Download

  • Access the feedback form for an event after deployment.
  • Submit feedback to rate the current event and help improve future events.
  • Download your certificate directly from the form after submission.

Certificate Repository

  • Go to the certificate repository page.
  • Search for certificates by name or event.
  • View and verify certificate authenticity.

API Reference

Certificate Template Editor

The system uses Konva.js to provide an interactive canvas for admins to position the sample name placeholder on certificate templates. This ensures that recipient names appear at the correct location on generated certificates.

Feedback Form QR Code

For each event, a unique QR code is generated and shown to the participants. Scanning this QR code directs users to the feedback form for that specific event.

Data Storage

  • Firebase Firestore: Stores certificate recipient lists, event details, and feedback responses.
  • Cloudinary: Used to upload and store certificate template images securely.

FAQ

Q: How do I report bugs?
A: Please open an issue on our GitHub repository.

Q: How do I generate certificates for a new event?
A: Log in to the admin panel, create a new event, upload a certificate template, and deploy feedback form.

Q: Can participants download their certificates directly?
A: Yes, after submitting feedback, participants can download their certificates from the provided button.

Q: How is certificate authenticity verified?
A: Each certificate has a unique Response ID and can be verified via the certificate repository page.

Q: What file formats are supported for certificate templates?
A: Common image formats such as PNG and JPEG are supported. But PNG is recommended for better quality.

Q: Where is my data stored?
A: Data is securely stored in Firebase Firestore and certificate templates are stored in Cloudinary.

Q: Who can access the admin panel?
A: Only authorized administrators with valid credentials can access the admin panel.

Usability Testing Results

The end-users were tasked to simply join the server, where the link was provided on the intern-group chat. After routing to the feedback form, all interns were asked to rate the User Interface of the Automated Certificate System. Upon successful entry, the users are given a link to a satire certificate with their name in it. In this process the certificates were all carefully inspected to check placement accuracy, and the test concludes.

All names in the certificates were placed correctly, and the users were able to download their certificates without any issues. The feedback form was also easy to navigate, and all participants were able to submit their feedback successfully.
